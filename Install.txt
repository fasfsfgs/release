Pré-requisitos
--------------

svn (de linha de comando) e mvn no PATH.
   
-- Qual versão do SVN?

Depende. Da versão 1.6 pra 1.7 o svn mudou o formato da cópia de trabalho, quebrando a compatibilidade. Um não entende o formato do outro -- mas o 1.7 tem um comando 'upgrade' que, como o próprio nome diz, faz o upgrade.

Então vc tem que descobrir qual é o svn que vc já usa, pra instalar o cliente de linha de comando correspondente. Como muito bem observou o nosso colega Lucivaldo, se vc usa o Eclipse Mindigo, vc está usando o svn 1.7 (vc sabia disso?).

Para verificar se o cliente de linha de comando casa com o do Eclipse, faça:

1. Abra um terminal na pasta de alguma cópia de trabalho.

2. Digite

       svn st
    
    Se as versões foram incompatíveis, ele vai dizer:

        svn: The path '.' appears to be part of a Subversion 1.7 or greater working copy.
        Please upgrade your Subversion client to use this working copy.
 

Instalação
----------

1. Crie o arquivo

       install.properties
    
   a partir de
    
       install.properties.example
       
   Altere o valor de 'install.dir' (no Windows, as contrabarras ('\') devem ser substuídas por "\\").
   
   Vc deve criar o diretorio de instalação(install.dir), pq o processo de instalação não o cria e não instala se ele não existir


2. Execute

       ./install     (Unix-like)
    
   ou
    
       install       (Windows)


3. Acrescente ${install.dir}/bin ao PATH.

4. [*** Windows only ***] Crie a variável de ambiente

       BAT=".bat"

   (necessário para a tarefa de Maven Tasks)
